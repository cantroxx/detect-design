# Quest Flow

## Campaign Structure

레퍼런스처럼 사건은 하나로 끝나지 않는다. CASE 01~05가 순서대로 열리고, 각 사건의 마지막 태도는 탐정 스타일과 캠페인 스탯으로 남는다.

- 기록형 탐정: 다음 사건에서 기록장, 숫자, 빈칸 단서가 먼저 열린다.
- 공감형 탐정: 다음 사건에서 친구의 제보와 대화 선택지가 먼저 열린다.
- 성장형 탐정: 다음 사건에서 확인 체크리스트가 먼저 열려 성급한 판단을 줄인다.

## CASE List

1. CASE 01: 사라진 칭찬 쿠폰
2. CASE 02: 도서관 대출 카드의 빈칸
3. CASE 03: 운동장 시간표의 두 이름
4. CASE 04: 익명 게시판의 이상한 댓글
5. CASE 05: 사라진 탐정 노트

## CASE 01: 사라진 칭찬 쿠폰

### 사건 발생

아침 조회 시간, 칭찬 쿠폰 상자의 숫자가 맞지 않는다. 몇몇 친구는 누군가 몰래 가져갔다고 생각한다. 플레이어는 디지털 탐정단이 되어 교실 곳곳을 살펴본다.

### 조사 순서

1. 쿠폰 상자 확인: 남은 쿠폰 수가 기록표와 다르다.
2. 기록표 확인: 어제 마지막 기록에는 24장이 남아 있었다.
3. 청소 당번표 확인: 발표 준비 모둠이 방과 후에 교실을 사용했다.
4. 발표 준비 메모 확인: 쿠폰 봉투를 발표 자료 봉투와 함께 옮겼다는 메모가 있다.

### 진실

쿠폰은 사라진 것이 아니라 발표 자료 봉투에 섞여 있었다. 누군가 일부러 가져간 사건이 아니다.

### 증거 연결

- 기록표 -> 쿠폰 상자: 숫자가 맞지 않는다는 사실 확인.
- 청소 당번표 -> 발표 준비 메모: 방과 후 교실을 쓴 팀 확인.
- 발표 준비 메모 -> 쿠폰 상자: 봉투가 섞인 원인 확인.

### 마지막 선택

- 선생님께 기록과 봉투를 함께 확인한다.
- 친구들에게 먼저 차분히 설명하고 함께 찾는다.
- 바로 모두에게 누가 실수했다고 말한다.

### 엔딩

- 꼼꼼한 탐정: 기록을 바탕으로 정확히 해결한다.
- 따뜻한 탐정: 친구가 부끄럽지 않게 함께 정리한다.
- 다시 확인하는 탐정: 성급한 말 때문에 다시 확인의 중요성을 배운다.

### 다음 사건 연결

CASE 01이 끝나면 사건 파일에는 해결 기록과 탐정 스타일이 남고 CASE 02가 열린다. 같은 구조로 CASE 05까지 진행한 뒤, 누적 스탯으로 캠페인 최종 엔딩을 결정한다.

## Campaign Endings

- 기록의 탐정단: 기록 확인과 다시 확인 수치가 높을 때.
- 마음을 잇는 탐정단: 마음 살피기 수치가 높을 때.
- 함께 해결하는 탐정단: 함께 해결 수치가 높을 때.
- 성장하는 탐정단: 실수도 있었지만 다시 확인하며 회복한 기본 성장 엔딩.
- 소문을 멈춘 탐정단: 확인 없는 말이 여러 번 퍼졌을 때 열리는 교훈 엔딩.
