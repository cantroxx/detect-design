# 디지털 탐정단: 사라진 칭찬 쿠폰

## 목표

초등학교 4학년이 즐길 수 있는 정보 추리 웹게임을 만든다. 아이들은 단서를 모으고, 기록을 비교하고, 증거를 연결하며, 친구를 탓하기보다 사실을 확인하고 함께 해결하는 방법을 배운다.

## 핵심 원칙

- 범인 찾기보다 원인 찾기.
- 처벌보다 회복과 약속.
- 짧은 문장, 명확한 선택지.
- 실제 브랜드, 실제 학교명, 실제 인물 없음.
- 이미지에는 글자를 넣지 않고, 모든 읽기 텍스트는 HTML UI로 렌더링.
- 배경은 imagegen으로 생성하고, 클릭 영역과 단서 카드는 코드로 관리.

## 현재 구현 범위

- 플레이 가능한 챕터 1개: 사라진 칭찬 쿠폰.
- 장소 1개: 4학년 교실.
- 프롤로그 3장: 사건 발생, 의심의 시작, 탐정단 임무.
- 핫스팟별 대화 이벤트 4개.
- 단서 4개: 쿠폰 상자, 기록표, 청소 당번표, 발표 준비 메모.
- 증거 연결 3개.
- 사건 보고서 작성 1개: 단서 연결 뒤 사실을 순서대로 정리해야 결말 선택이 열린다.
- 엔딩 3종: 꼼꼼한 확인, 따뜻한 해결, 성급한 발표.
- 사건 파일 허브: CASE 01 기록, CASE 02 예고, CASE 03 잠금 상태를 보여준다.
- 탐정 스타일 기록: CASE 01의 마지막 선택에 따라 기록형, 공감형, 성장형 탐정으로 누적된다.

## 레퍼런스 재검토 결과

- 레퍼런스는 단일 사건이 아니라 여러 챕터를 순차 해금하는 캠페인 구조다.
- 각 챕터는 `objectives`, `boardDone`, `finishChapter`, `next chapter` 흐름을 갖는다.
- 엔딩은 바로 끝나는 화면이 아니라 결정 기록, 능력치, 다음 챕터 예고를 남긴다.
- 따라서 이 프로젝트도 CASE 01 완결 뒤 탐정 스타일과 다음 사건의 첫 단서가 달라지는 방향으로 확장한다.

## 스토리 개편 방향

- 플레이어가 시스템을 먼저 보는 것이 아니라 사건 상황을 먼저 목격한다.
- 단서 클릭은 즉시 보상이 아니라 짧은 대화 이벤트를 거친다.
- 단서는 `탐정 노트 갱신`으로 기록되어 수집감이 생기게 한다.
- 증거 보드는 모든 단서가 모인 뒤 `BOARD READY` 상태로 해금된다.
- 증거 연결 뒤 바로 끝나지 않고, `기록 → 오늘 확인 → 가능한 이유 → 말하는 태도` 순서로 보고서를 작성한다.
- 결말 선택은 보고서 제출 뒤 열리며, 정답 선택이 아니라 사실을 말하는 태도 선택으로 구성한다.
- 사건이 끝나면 CASE LOG와 DETECTIVE STYLE을 남겨 다음 사건의 분기 근거로 사용한다.

## 학습 포인트

- 기록을 먼저 확인하기.
- 증거 없이 단정하지 않기.
- 친구의 마음을 생각하며 말하기.
- 문제 해결 과정을 정리하기.
